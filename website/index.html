Indexarr

A Community-Powered Torrent Index

Users upload torrents. Nodes sync with each other. Everyone gets the full index. Indexarr is a decentralized torrent indexer where every contributor grows the catalog for the entire network — no central authority, no accounts, no tracking. Fully compatible with the *arr ecosystem.

Download Live Demo Learn More

Features

Community Uploads

Anyone can upload .torrent files or paste magnet links to contribute to the shared index. Every upload is automatically classified and made searchable across the entire network. No accounts, no tracking.

Network-Wide Sync

Uploads on any node are automatically shared across the entire network via gossip protocol. Run a node and you get the full index — every contributor's uploads, not just your own.

Smart Classification

Every torrent is parsed and classified automatically — content type, resolution, codec, source, and matched against TMDB. Search by category, filter by quality, find what you need.

*arr Ecosystem

Full Torznab/Newznab API with 30 categories. Plug directly into Sonarr, Radarr, Lidarr, Readarr, and Prowlarr. Your self-hosted indexer, fed by the community.

No Tracking

No user accounts, no IP logging, no upload history, no telemetry. Searches happen locally on your node. The network only shares torrent metadata — never who searched for what.

DHT Discovery

Optionally crawl the BitTorrent DHT to automatically discover and index torrents from the public network. Supplement community uploads with passive discovery.

How It Works

1

Upload

Contributors across the network upload .torrent files and magnet links to their nodes.

2

Classify

Each node automatically parses and classifies uploads — content type, quality, codec — ready for search.

3

Sync

Gossip protocol spreads every upload across the network. Your node gets everyone else's contributions automatically.

4

Search

The full community index is searchable on your node — via the web UI, REST API, or Torznab for your *arr stack.

Docker Recommended

The easiest way to run Indexarr. Available on GitHub Container Registry.

Quick Start

docker compose up -d
# Access the UI at http://localhost:8080

Docker Compose

services:
  indexarr:
    image: ghcr.io/ausagentsmith-org/indexarr-rs:latest
    container_name: indexarr
    ports:
      - "8080:8080"
      - "6881-6884:6881-6884/udp"
      - "6895:6895/udp"
    volumes:
      - indexarr-data:/app/data
    environment:
      - INDEXARR_DB_URL=postgres://indexarr:indexarr@postgres:5432/indexarr
      - INDEXARR_SYNC_ENABLED=true
    restart: unless-stopped

volumes:
  indexarr-data:

Source

Build from source. Requires Rust 1.85+ and Node 22+ (for the UI).

Clone & Build

git clone https://github.com/AusAgentSmith-org/indexarr-rs.git
cd indexarr-rs

# Build the UI
cd ui && npm install && npm run build && cd ..

# Build the binary
cargo build --release

Run

INDEXARR_DB_URL=postgres://indexarr:indexarr@localhost/indexarr \
  ./target/release/indexarr --all

# Or run individual workers
./target/release/indexarr --workers http_server,dht_crawler

Run tests

cargo test --workspace
